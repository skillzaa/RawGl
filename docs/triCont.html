TriCont (Tri Container Component)

RawGL provides you with 3 core components which shall do all the drawing. These components are:
  1. BaseComp (Base Component).
  2. TriComp (Tri Component).
  3. TriContComp (Tri Container Component).

All the above three components has just one purpose and that is to draw Triangles on the canvas (which is the main objective of webGl).

All of the above Components (component is called comp in these documents) implements IGPUComp interface. It is called GPU comp since (in WebGl) the code runs in GPU and not on CPU.

Common features of All 3 Comps

All the components at the time of creation do not hold any data to draw.

All these comps gets input data through "addTri" method using which we feed it one triangle at a time.

After creation each component needs to be initialized. We call the init method and give it webGl context.

If we change the properties of these comps then we need to call "update" method for these changes to take effect.

Lets look at each component in detail

BaseComp:

This is the base component for rest of the two components.

The main difference that BaseComp has from rest of the two comps (TriComp and TriContComp) is that the BaseComp use WebGl coordinate system (which is from -1 to 1) where as the TriComp and TriContComp uses coordinate system from zero to hundred (0 percent to 100 percent).

TriComp

As stated above the only difference between this comp and BaseComp is that this comp takes screen coordinates in percentages from 0 to 100 (we can also go negative). This is much easy since calculation from -1 to 1 as is a norm in webGl are difficult.

TriContComp

TriContComp introduces the concept of x, y, width and height.

The "Tri Container Comp" is the main comp that we will use in this framework. Unlike BaseComp and TriComp the TriContComp just do not draw triangles onto the screen but contain them inside a box defined by the x, y, width and height properties.

We can move all the contents of this container by changing its x or y and we can all change the sizes by changing width and height properties (properties are called props)

We can make TriContComp stretch and cover the whole screen / canvas and we can also layer them on top of each other.

I foresee that TriContComp will be the main comp used for display and managing content.

Final Comps Comparison

Shapes drawn using BaseComp and TriComp are drawn on the main canvas and not inside a container. The shapes draw by TriContComp are restricted to the width and height of TriContComp.

To move shapes created by BaseComp and TriComp we have to move each vertex individually however incase of shapes drawn by TriContComp we can simply change the x and y value of the comp and all the shapes will move.

To change shapes width and height created by BaseComp and TriComp we have to move each vertex individually however incase of shapes drawn by TriContComp we can simply change the width and height value of the comp and all the shapes will widen/shorten etc.

How to add Shapes to these Comps

The primery source is to add one triangle at a time to any of the core comps using "addTri" method. This method takes the x and y values for all the 3 vertex of a triangle and also the color of the triangle in RGB format.

A easy method is to use the "comp.add" object which has some basic Primtive shapes that can be added to each comp.
